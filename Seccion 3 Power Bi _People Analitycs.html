1. Fórmulas DAX para Métricas Clave en People Analytics

DAX (Data Analysis Expressions) es un lenguaje de fórmulas especialmente diseñado para el análisis de datos en Power BI. Esta potente herramienta nos permite crear métricas personalizadas y complejas que van mucho más allá de los simples conteos o promedios, permitiéndonos extraer insights profundos sobre el capital humano de la organización.

La principal fortaleza de DAX en el contexto de People Analytics radica en su capacidad para:

  • Establecer relaciones entre diferentes dimensiones de datos de empleados
  • Crear cálculos dinámicos que responden al contexto (filtros, segmentaciones)
  • Implementar lógica condicional compleja para escenarios de RRHH
  • Realizar análisis temporales sofisticados (comparativas año contra año, tendencias)
  • Calcular métricas agregadas a diferentes niveles organizacionales

Consideración importante

Aunque DAX puede parecer intimidante al principio, dominar incluso un conjunto básico de funciones te permitirá crear métricas de RRHH significativamente más valiosas. Recomendamos un enfoque gradual, comenzando con fórmulas simples e ir incrementando la complejidad a medida que te sientas más cómodo con la sintaxis.

Contexto de Filtro en People Analytics

El contexto de filtro es fundamental para entender cómo DAX evalúa las expresiones. Representa el conjunto de filtros aplicados a tus datos en un momento determinado.

En análisis de RRHH, esto es crucial cuando calculas métricas como tasas de rotación por departamento o niveles de engagement por ubicación.

La función CALCULATE() te permite modificar este contexto, permitiéndote responder preguntas como: "¿Cuál sería la tasa de rotación si solo consideramos empleados con más de 2 años de antigüedad?"

Dominar el contexto de filtro te permitirá crear análisis segmentados precisos para diferentes unidades de negocio, demografías o cohortes de empleados.

Inteligencia de Tiempo para Análisis de RRHH

Las funciones de inteligencia de tiempo en DAX son esenciales para análisis de tendencias en métricas de RRHH, permitiendo comparativas periodo a periodo y análisis de evolución.

Funciones como SAMEPERIODLASTYEAR, DATEADD, y PARALLELPERIOD te permiten comparar la rotación actual con la del año anterior, ver la evolución del engagement o analizar tendencias de contratación.

Para People Analytics, esto es crítico al evaluar el impacto de programas de retención, cambios en políticas de compensación o iniciativas de cultura organizacional.

Siempre utiliza una tabla de calendario dedicada con jerarquías claras (año > trimestre > mes) para maximizar la eficacia del análisis temporal en tus dashboards de RRHH.

1. Métricas de Headcount y Demografía ▼

Las métricas de headcount son fundamentales para cualquier análisis de fuerza laboral, permitiendo entender la composición y distribución de tu personal a lo largo del tiempo y diferentes dimensiones organizacionales.

1.1 Headcount a Fecha Específica

Esta medida calcula el número de empleados activos en una fecha determinada, considerando fechas de contratación y terminación:

Headcount a Fecha = 
CALCULATE(
    COUNTROWS('Tabla Empleados'),
    'Tabla Empleados'[Fecha Contratación] <= MAX('Tabla Calendario'[Fecha]),
    OR(
        ISBLANK('Tabla Empleados'[Fecha Terminación]),
        'Tabla Empleados'[Fecha Terminación] > MAX('Tabla Calendario'[Fecha])
    )
)

1.2 Headcount Promedio en Período

Calcula el headcount promedio durante un período, útil para análisis de productividad y planificación de capacidad:

Headcount Promedio = 
AVERAGEX(
    VALUES('Tabla Calendario'[Fecha]),
    CALCULATE(
        COUNTROWS('Tabla Empleados'),
        'Tabla Empleados'[Fecha Contratación] <= 'Tabla Calendario'[Fecha],
        OR(
            ISBLANK('Tabla Empleados'[Fecha Terminación]),
            'Tabla Empleados'[Fecha Terminación] > 'Tabla Calendario'[Fecha]
        )
    )
)

1.3 Índice de Diversidad

Calcula el porcentaje de empleados que pertenecen a grupos diversos, con posibilidad de segmentación:

% Diversidad de Género = 
DIVIDE(
    CALCULATE(
        COUNTROWS('Tabla Empleados'),
        'Tabla Empleados'[Género] = "Mujer",
        'Tabla Empleados'[Activo] = TRUE
    ),
    CALCULATE(
        COUNTROWS('Tabla Empleados'),
        'Tabla Empleados'[Activo] = TRUE
    ),
    0
)

1.4 Distribución por Antigüedad

Crea una categorización de empleados basada en su antigüedad, útil para análisis de cohortes:

Antigüedad Categorizada = 
VAR AntigüedadAños = 
    DATEDIFF('Tabla Empleados'[Fecha Contratación], TODAY(), YEAR)
RETURN
    SWITCH(
        TRUE(),
        AntigüedadAños < 1, "< 1 año",
        AntigüedadAños < 3, "1-3 años",
        AntigüedadAños < 5, "3-5 años",
        AntigüedadAños < 10, "5-10 años",
        "10+ años"
    )

Consejo práctico

Para análisis demográficos precisos, es fundamental mantener una tabla de "foto" de empleados por fecha. Esto puede crearse en Power Query duplicando la tabla de empleados para cada fecha relevante, permitiendo análisis históricos precisos incluso cuando la composición del personal cambia.

2. Métricas de Rotación y Retención ▼

Las métricas de rotación son quizás las más demandadas en People Analytics, ya que tienen un impacto directo en los costos, la productividad y la cultura organizacional. DAX nos permite crear cálculos sofisticados que van más allá de las fórmulas básicas.

2.1 Tasa de Rotación Voluntaria

Esta fórmula calcula el porcentaje de empleados que dejaron voluntariamente la organización en un período específico:

Tasa Rotación Voluntaria = 
VAR SalidasVoluntarias = 
    CALCULATE(
        COUNTROWS('Tabla Empleados'),
        'Tabla Empleados'[Motivo Salida] IN {"Renuncia Voluntaria", "Mejor Oferta", "Relocalización"},
        'Tabla Empleados'[Fecha Terminación] >= MIN('Tabla Calendario'[Fecha]),
        'Tabla Empleados'[Fecha Terminación] <= MAX('Tabla Calendario'[Fecha])
    )
VAR HeadcountPromedio = 
    AVERAGEX(
        VALUES('Tabla Calendario'[Fecha]),
        CALCULATE(
            COUNTROWS('Tabla Empleados'),
            'Tabla Empleados'[Fecha Contratación] <= 'Tabla Calendario'[Fecha],
            OR(
                ISBLANK('Tabla Empleados'[Fecha Terminación]),
                'Tabla Empleados'[Fecha Terminación] > 'Tabla Calendario'[Fecha]
            )
        )
    )
RETURN
    DIVIDE(SalidasVoluntarias, HeadcountPromedio, 0)

2.2 Tasa de Supervivencia por Cohorte

Esta medida avanzada calcula el porcentaje de empleados de una cohorte de contratación que siguen en la empresa después de un período específico:

Tasa Supervivencia a 12 Meses = 
VAR TamañoCohorte = 
    CALCULATE(
        COUNTROWS('Tabla Empleados'),
        'Tabla Empleados'[Año-Mes Contratación] = SELECTEDVALUE('Tabla Cohortes'[Año-Mes])
    )
VAR ActivosDespuesDe12Meses = 
    CALCULATE(
        COUNTROWS('Tabla Empleados'),
        'Tabla Empleados'[Año-Mes Contratación] = SELECTEDVALUE('Tabla Cohortes'[Año-Mes]),
        OR(
            ISBLANK('Tabla Empleados'[Fecha Terminación]),
            'Tabla Empleados'[Fecha Terminación] > EDATE('Tabla Empleados'[Fecha Contratación], 12)
        )
    )
RETURN
    DIVIDE(ActivosDespuesDe12Meses, TamañoCohorte, 0)

2.3 Proyección de Riesgo de Rotación

Una fórmula predictiva que combina múltiples factores para estimar el riesgo de rotación de los empleados:

Índice de Riesgo de Rotación = 
VAR RiesgoBase = 0.05
VAR FactorAntiguedad = 
    IF(
        [Antigüedad en Años] < 1, 
        2,
        IF(
            [Antigüedad en Años] < 2,
            1.5,
            IF(
                [Antigüedad en Años] > 5,
                0.7,
                1
            )
        )
    )
VAR FactorSatisfaccion = 
    SWITCH(
        [Nivel Satisfacción],
        "Muy bajo", 2.5,
        "Bajo", 1.8,
        "Medio", 1,
        "Alto", 0.7,
        "Muy alto", 0.4,
        1
    )
VAR FactorSalario =
    IF(
        [Compensación vs Mercado] < -10,
        1.8,
        IF(
            [Compensación vs Mercado] < -5,
            1.3,
            IF(
                [Compensación vs Mercado] > 5,
                0.8,
                1
            )
        )
    )
VAR FactorCarga =
    IF(
        [Horas Extra Mensuales] > 20,
        1.6,
        IF(
            [Horas Extra Mensuales] > 10,
            1.2,
            1
        )
    )
RETURN
    RiesgoBase * FactorAntiguedad * FactorSatisfaccion * FactorSalario * FactorCarga

Recomendación avanzada

Para mejorar el modelo predictivo de rotación, considera incorporar variables como el tiempo desde la última promoción, la distancia entre hogar y oficina, y el nivel de satisfacción con el gerente directo. La combinación de estos factores puede mejorar significativamente la precisión del modelo.

3. Métricas de Compensación y Equidad ▼

El análisis de compensación es una parte crucial del People Analytics, no solo para garantizar competitividad en el mercado sino también para asegurar la equidad interna y cumplir con regulaciones cada vez más estrictas.

3.1 Brecha Salarial Ajustada

Esta medida calcula la diferencia porcentual en salarios entre grupos demográficos, ajustada por factores como rol, nivel, y ubicación:

Brecha Salarial Ajustada = 
VAR SalarioPromedioGrupo1 = 
    CALCULATE(
        AVERAGE('Tabla Empleados'[Salario Anual]),
        'Tabla Empleados'[Grupo Demográfico] = "Grupo1"
    )
VAR SalarioPromedioGrupo2 = 
    CALCULATE(
        AVERAGE('Tabla Empleados'[Salario Anual]),
        'Tabla Empleados'[Grupo Demográfico] = "Grupo2"
    )
RETURN
    DIVIDE(
        SalarioPromedioGrupo1 - SalarioPromedioGrupo2,
        SalarioPromedioGrupo2,
        0
    )

3.2 Compa-Ratio (Ratio de Competitividad)

Compara el salario actual con el punto medio del rango salarial establecido para cada posición:

Compa-Ratio = 
DIVIDE(
    'Tabla Empleados'[Salario Actual],
    RELATED('Tabla Rangos Salariales'[Punto Medio]),
    0
)

3.3 Incremento Salarial Tendencia

Calcula el incremento salarial promedio año tras año para diferentes grupos o departamentos:

% Incremento Salarial YoY = 
VAR SalarioActual = 
    CALCULATE(
        AVERAGE('Tabla Empleados'[Salario Anual])
    )
VAR SalarioAñoAnterior = 
    CALCULATE(
        AVERAGE('Tabla Empleados'[Salario Anual]),
        SAMEPERIODLASTYEAR('Tabla Calendario'[Fecha])
    )
RETURN
    IF(
        SalarioAñoAnterior > 0,
        DIVIDE(
            SalarioActual - SalarioAñoAnterior,
            SalarioAñoAnterior,
            0
        ),
        BLANK()
    )

Consideración legal

Al realizar análisis de equidad salarial, asegúrate de consultar con el departamento legal sobre las regulaciones específicas de cada región donde opera la empresa. Muchos países tienen requisitos y definiciones diferentes para lo que constituye discriminación salarial.

1. Estructura Óptima de Datos para RRHH ▼

La estructura de tu modelo de datos es el fundamento de todo análisis efectivo. Para People Analytics, hay consideraciones específicas que mejorarán significativamente tus resultados.

Dimensiones Clave para People Analytics

  • Empleado como dimensión central: Crea una tabla de dimensiones de empleados con un identificador único para cada persona, conectada a las tablas de hechos como rendimiento, ausencias, compensación, etc.
  • Estructura organizacional jerárquica: Implementa relaciones de "muchos a uno" para representar la jerarquía organizacional (empleado → manager → departamento → división)
  • Tabla de calendario robusta: Incluye jerarquías de tiempo, año fiscal vs. calendario, días laborables vs. no laborables
  • Dimensión de "punto en el tiempo": Fundamental para análisis históricos precisos, permitiendo "fotografías" de la organización en momentos específicos

Consejo avanzado

Utiliza el patrón "Slowly Changing Dimension" (SCD) Tipo 2 para tu tabla de empleados. Esto te permitirá rastrear cambios en atributos clave como departamento, nivel salarial, ubicación o manager a lo largo del tiempo, mejorando significativamente tu capacidad de analizar tendencias históricas.

Ejemplo: Estructura de un Modelo de Datos para People Analytics

// Ejemplo de relaciones clave en el modelo
// Tabla Central de Empleados
Empleados {
  EmpleadoID [PK]
  NombreCompleto
  FechaInicioRegistro (para SCD Tipo 2)
  FechaFinRegistro (para SCD Tipo 2)
  EsRegistroActual (Booleano)
  ...atributos adicionales
}

// Tablas de Dimensiones
Departamentos {
  DepartamentoID [PK]
  NombreDepartamento
  DirectorID (FK a Empleados)
}

Ubicaciones {
  UbicacionID [PK]
  Ciudad
  País
  Región
}

// Tablas de Hechos
Compensaciones {
  EmpleadoID [FK]
  FechaEfectiva
  TipoCompensación
  Monto
}

Ausencias {
  AusenciaID [PK]
  EmpleadoID [FK]
  FechaInicio
  FechaFin
  TipoAusencia
  Duración
}

Evita estos errores comunes

Uno de los errores más frecuentes en modelos de datos de RRHH es no manejar adecuadamente los cambios organizacionales a lo largo del tiempo. Si simplemente sobrescribes el departamento de un empleado cuando cambia, perderás la capacidad de analizar historiales de rotación por departamento o el impacto de reorganizaciones.

3. Gestión de Seguridad y Privacidad de Datos ▼

Los datos de RRHH son inherentemente sensibles, conteniendo información personal y confidencial de los empleados. Implementar las medidas de seguridad adecuadas es crucial para mantener la confianza y cumplir con normativas.

Control de Acceso por Roles (RBAC)

Power BI permite implementar controles de acceso granulares que son esenciales para datos de RRHH:

Niveles de Acceso Recomendados para People Analytics

Ejecutivos C-Level

Acceso a dashboards ejecutivos con tendencias agregadas, métricas estratégicas y KPIs críticos de toda la organización.

Directores y Gerentes

Acceso a métricas de sus departamentos con capacidad de comparación con benchmarks de la organización (sin datos de individuos en otros departamentos).

Profesionales RRHH

Acceso detallado a datos relacionados con sus áreas funcionales (reclutamiento, compensación, etc.) con personalización por región o unidad de negocio.

Técnicas de Seguridad en Power BI para Datos de RRHH

  • Seguridad a nivel de fila (RLS): Configura filtros dinámicos basados en la identidad del usuario para limitar datos visibles
  • Anonimización estratégica: Para análisis de diversidad o compensación, donde grupos pequeños podrían comprometer el anonimato
  • Agregación mínima: Establece reglas como "no mostrar datos para grupos de menos de 5 empleados"
  • Auditoría de uso: Monitoriza quién accede a qué informes y cuándo

Regulaciones importantes

Dependiendo de las regiones donde opera tu empresa, considera normativas como GDPR (Europa), CCPA (California), LGPD (Brasil) u otras leyes locales de protección de datos. Consulta siempre con el equipo legal sobre los requisitos específicos aplicables a los datos de empleados.

Ejemplo: Configuración de Seguridad a Nivel de Fila en DAX

// Ejemplo de RLS para acceso de gestores solo a sus equipos directos
// (crear en la pestaña de Roles en Power BI Desktop)

[EmpleadoID] = USERNAME() || 
CONTAINS(
    FILTER(
        'Tabla Empleados',
        'Tabla Empleados'[ManagerID] = USERNAME()
    ),
    [EmpleadoID],
    'Tabla Empleados'[EmpleadoID]
)
4. Implementación y Adopción Exitosa ▼

Desarrollar dashboards técnicamente sólidos es solo la mitad del camino. Para que un proyecto de People Analytics tenga éxito, necesitas asegurar su adopción y uso efectivo en toda la organización.

Estrategias para la Adopción por Usuarios de RRHH

  1. Identificar y resolver casos de uso críticos: Comienza con problemas específicos que afectan directamente a los resultados del negocio (ej. reducción de rotación en posiciones críticas)
  2. Personalizar para diferentes audiencias: No uses "talla única" para tus reportes - cada nivel necesita perspectivas diferentes:
    • Ejecutivos: Métricas de alto nivel con benchmarks externos
    • Gerentes: Tendencias departamentales con alertas de problemas
    • HR Business Partners: Datos detallados para intervenciones específicas
  3. Programa de champions: Identifica usuarios entusiastas en cada departamento para convertirlos en evangelizadores
  4. Ciclos de feedback estructurados: Reúnete regularmente con usuarios clave para entender qué funciona y qué no

Crear narrativa efectiva

Los mejores dashboards de People Analytics cuentan una historia. Utiliza anotaciones, tooltips informativos y páginas organizadas en una secuencia lógica que guíe a los usuarios a través de un análisis coherente, no solo una colección de gráficos.

5. Integraciones y Fuentes de Datos para People Analytics ▼

Los mejores proyectos de People Analytics combinan datos de múltiples fuentes para proporcionar una visión holística del capital humano. Power BI ofrece poderosas capacidades de integración que permiten unificar estos datos diversos.

Fuentes de Datos Esenciales para People Analytics

SISTEMAS CORE DE RRHH

  • HRIS/HCM: Datos demográficos, organizacionales y de compensación (Workday, SuccessFactors, etc.)
  • ATS: Datos de reclutamiento, tiempo de contratación, fuentes de candidatos
  • LMS: Datos de formación, certificaciones, desarrollo de habilidades

SISTEMAS OPERACIONALES

  • CRM/Ventas: Cruciales para vincular métricas de empleados con resultados de negocio
  • ERP/Finanzas: Para análisis de productividad, costos por empleado, ROI de RRHH
  • Gestión de proyectos: Para medir utilización, capacidad y eficacia de equipos

DATOS NO ESTRUCTURADOS

  • Encuestas de compromiso: Métricas de satisfacción, eNPS, comentarios de empleados
  • Herramientas de colaboración: Teams, Slack, email (análisis de redes organizacionales)
  • Evaluaciones 360°: Feedback multidimensional sobre desempeño

FUENTES EXTERNAS

  • Datos de mercado laboral: Tasas de desempleo, salarios de mercado por posición
  • Benchmarks industriales: Rotación promedio del sector, costos de contratación
  • Indicadores económicos: Para contextualizar tendencias de RRHH

Estrategias de Integración en Power BI

  1. Data Warehouse centralizado: Ideal para organizaciones grandes, permite ETL complejo y governance sólido
  2. Power BI Dataflows: Solución intermedia que permite transformaciones y reutilización de datos sin un DW dedicado
  3. Conexiones directas: Para organizaciones pequeñas o proyectos iniciales, conectando directamente a APIs o exportaciones

Consejo de arquitectura

Para proyectos de People Analytics maduros, implementa un enfoque de "medallion architecture" (bronze, silver, gold) que separe datos brutos, datos procesados y métricas calculadas. Esto facilita la governance, la documentación y la reutilización de transformaciones complejas.

Ejemplo: Arquitectura de Datos para People Analytics

// Enfoques de actualización recomendados por tipo de dato

// 1. Datos demográficos e históricos (carga incremental)
let
    Fuente = #"Tabla Empleados",
    NuevosRegistros = Table.SelectRows(
        Fuente, 
        each [FechaModificación] > #"Última Actualización"
    ),
    Combinado = Table.Combine({#"TablaDW", NuevosRegistros})
in
    Combinado

// 2. Datos transaccionales (append-only)
let
    Fuente = #"Tabla Ausencias",
    NuevosPeriodo = Table.SelectRows(
        Fuente,
        each [FechaCreación] > #"Última Actualización"
    ),
    Combinado = Table.Combine({#"TablaDW", NuevosPeriodo})
in
    Combinado

// 3. Datos snapshot (sobrescritura completa para pequeños datasets)
let
    Fuente = #"OrganizationChart",
    Actualizado = Fuente  // Sobrescribe completamente
in
    Actualizado
